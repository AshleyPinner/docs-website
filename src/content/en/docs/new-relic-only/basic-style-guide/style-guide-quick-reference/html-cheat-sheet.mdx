NR ONLY

This document lists most of the "house" HTML classes used on [**docs.newrelic.com**](https://docs.newrelic.com). You can use the various HTML snippets in this document to easily copy and paste particular styles into your document.

## Headings

For when to use a particular heading, see [Levels of headings](/docs/new-relic-only/basic-style-guide/writing-guidelines/levels-headings).

For H2s:

```
<h2 id="INSERT_ID">INSERT_HEADER_TITLE</h2>
```

For H3s:

```
<h3 class="freq-link" id="INSERT_ID">INSERT_HEADER_TITLE</h3>
```

## Clamshells (collapsers)

For when to use collapsers, see [Clamshells](/docs/new-relic-only/basic-style-guide/writing-guidelines/clamshells).

For a basic clamshell:

```
<dl class="clamshell-list">

    <dt id="INSERT_ID">INSERT_HEADER_TITLE</dt>
    <dd>
        <p>INSERT_DESCRIPTION</p>
    </dd>

</dl>
```

For a special example clamshell ([example](/docs/new-relic-only/basic-style-guide/writing-guidelines/example-clamshell)):

```
<dl class="example-box">

    <dt id="INSERT_ID">INSERT_HEADER_TITLE</dt>
    <dd>
        <p>INSERT_DESCRIPTION</p>
    </dd>

</dl>
```

To add additional items within the list of clamshells, add `<dt>` and `<dd>` tags between the `<dl>` tags:

```
<dt id="INSERT_ID">INSERT_HEADER_TITLE</dt>
<dd>
    <p>INSERT_DESCRIPTION</p>
</dd>
```

## Callouts

For when to use a particular callout, see [Callouts](/docs/new-relic-only/basic-style-guide/writing-guidelines/callouts).

For tips (grey) callouts:

```
<div class="callout-tip">
    <p>INSERT_CALLOUT_TEXT</p>
</div>
```

For important (yellow) callouts:

```
<div class="callout-important">
    <p>INSERT_CALLOUT_TEXT</p>
</div>
```

For caution (red) callouts:

```
<div class="callout-caution">
    <p>INSERT_CALLOUT_TEXT</p>
</div>
```

For permissions callouts:

```
<div class="callout-permissions">
    <p><b>INSERT_CALLOUT_TEXT</b></p>
</div>
```

For pricing callouts (rarely used):

```
<div class="callout-pricing">
    <p>INSERT_CALLOUT_TEXT</p>
</div>
```

## Lists

For more information on lists, see [Lists](/docs/new-relic-only/basic-style-guide/writing-guidelines/lists).

For unordered (bulleted) lists:

```
<ul>
    <li>INSERT_LIST_ITEM_TEXT</li>
</ul>
```

For ordered (numbered) lists:

```
<ol>
    <li>INSERT_LIST_ITEM_TEXT</li>
</ol>
```

## Code

For when to use a particular type of code tag, see [Code examples](/docs/new-relic-only/basic-style-guide/writing-guidelines/code-examples).

For multi-line code blocks:

```
<pre>YOUR_CODE_HERE</pre>
```

For inline code:

```
<code>YOUR_CODE_HERE</code>
```

To highlight user-supplied values:

```
<var>YOUR_CODE_HERE</var>
```

To draw attention to part of a large code sample:

```
<mark>YOUR_CODE_HERE</mark>
```

For syntax highlighting:

```
<pre class="highlight">YOUR_CODE_HERE</pre>
```

## Tables

Most tables on the site use a basic HTML [table](/docs/new-relic-only/basic-style-guide/writing-guidelines/tables):

```
<table>
    <thead>
        <tr>
            <th style="width:200px">COLUMN_1_HEADER</th>
            <th>COLUMN_2_HEADER</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>COLUMN_1_DATA</td>
            <td>COLUMN_2_DATA</td>
        </tr>
    </tbody>
</table>
```

However, config docs use a simpler specs table ([example](/docs/infrastructure/new-relic-infrastructure/configuration/configure-infrastructure-agent#proxy)):

```
<table class="specs">
    <tbody>
        <tr>
            <th>Type</th>
            <td>INSERT_DATA_TYPE</td>
        </tr>
        <tr>
            <th>Default</th>
            <td>INSERT_DEFAULT_VALUE</td>
        </tr>
    </tbody>
</table>
```

## Links

For when to use hyperlinks, see [Hyperlinks](/docs/new-relic-only/basic-style-guide/writing-guidelines/hyperlinks).

For a basic link:

```
<a href="INSERT_URL">INSERT_LINK_TEXT</a>
```

For an anchor link:

```
<a href="#INSERT_ANCHOR_ID">INSERT_LINK_TEXT</a>
```

## Watermarks

For when to use watermarks, see [Watermarks](/docs/new-relic-only/tech-writer-style-guide/design-specifications/classes-styles#watermarks).

For a beta:

```
<div id="watermark">BETA</div>
```

For internal documents only available to people at New Relic:

```
<div id="watermark">NR ONLY</div>
```

For a legacy product (like New Relic Servers):

```
<div id="watermark">Legacy</div>
```

For a limited release:

```
<div id="watermark">Limited <br /> release</div>
```
