NR ONLY

Tables help you organize information to make it easier for readers to skim. However, if your table becomes too long, follow the ["five to nine" guideline](/docs/new-relic-only/basic-style-guide/basic-style-guide/writing-document#organization) and consider restructuring the information in other ways.

You can use HTML, Markdown, or the WYSIWYG to **add** tables to your docs. You must use HTML to **edit** tables. Use sentence case for text in every table column unless it is a specific file name or code value that requires lower case or all caps.

## Table template

This is a basic table template.

```
<table>
<thead>
  <tr>
    <th style="width:200px"><b>Item</b></th>
    <th><b>Requirements</b></th>
  </tr>
</thead>
<tbody>
  <tr>
    <td>Item name</td>
    <td>Description of item</td>
  </tr>
</tbody>
</table>
```

Additional options:

<Table>
  <thead>
    <tr>
      <th style={{ width: "200px" }}>
        If you want to...
      </th>

      <th>
        Do this
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        Shade every other row
      </td>

      <td>
        Use `<table class="alternate">`.
      </td>
    </tr>

    <tr>
      <td>
        Create a table within a clamshell
      </td>

      <td>
        Use `<table class="spec">`.
      </td>
    </tr>

    <tr>
      <td>
        Highlight row in gray on hover
      </td>

      <td>
        Add the class `table-hover` to the `<table>` element.
      </td>
    </tr>

    <tr>
      <td>
        Specify individual column widths
      </td>

      <td>
        Set `style="width:###px;"` in the header row. In general, for two-column tables on the Docs site:

        1. Set the first column as 200 pixels: `<th style="width:200px">`.
        2. To allow the table to adjust automatically to the remaining page width, do not set a width for the second column.
      </td>
    </tr>
  </tbody>
</Table>

## Tables to describe UI functions

When using tables to describe available functions, give readers a roadmap. They may know what they want to find or accomplish, but they may not know what it's called or how to do it.

For example, to provide a list of features for a particular UI page, label the header's first column **If you want to...** and label the second column **Do this**.

## WYSIWYG table formatting

To insert a table using the Docs site's WYSIWYG editor:

1. Select the **Table tool**.
2. Specify the number of rows and columns.
3. Enter the number of rows and columns.
4. Clear the width.
5. Select **OK**.

To access the table settings:

1. Right-click the table.
2. Select **Table properties**.

To set the column width:

1. Right-click the first cell in the column (the header cell).
2. Select **Cells > Cell properties**.
3. Set the **Width** in pixels.
4. Select **OK**.
