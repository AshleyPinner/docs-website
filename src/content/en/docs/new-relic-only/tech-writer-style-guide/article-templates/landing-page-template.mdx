This doc provides an example of [landing page HTML](#template) to copy and paste into the Page Contents field when [working with your landing page](/docs/new-relic-only/drupal-configuration/content-types/working-landing-pages), as well as some important info to keep in mind when [updating the landing page components](#update-components) with your info.

## Update landing page components

Ensure you update the following components in the template:

<CollapserGroup>
  <Collapser
    id="featured"
    title="Featured content format"
  >
    The template demos two different formats for the featured content columns: text wrapped around the icon and icon flush left in its own column.
  </Collapser>

  <Collapser
    id="add-media"
    title="Media (image or video)"
  >
    Add an image via the atom pane, per normal, but remove the comments the Scald module creates.

    Add a Wistia video per normal.
  </Collapser>

  <Collapser
    id="colors"
    title="Product colors for icons"
  >
    To update icon colors, add the relevant product colors with [product color classes](/docs/new-relic-only/basic-style-guide/writing-guidelines/color-design-specs#product_colors) for color (text and fontawesome icons).
  </Collapser>

  <Collapser
    id="link-button"
    title="Link button to standard menu page"
  >
    The link button at the bottom of the landing has an `href` as the current URL with a parameter telling the menu page to load the standard menu table of contents. The HTML looks like this with your values substituted for the product colors and the URL the landing page is located. Note that the `product-wrap` class must be present on the parent `<div>`.

    ```
    <div class="row">
      <div class="col col-md-12 add-margin text-center product-wrap">
        <a class="btn alerts" href="/docs/alerts?toc=true" role="button">View all Alerts docs</a>
      </div>
    </div>
    ```
  </Collapser>
</CollapserGroup>
